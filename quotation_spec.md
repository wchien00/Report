## 💻 技術規格與環境 (Technical Stack)

本專案採用**輕量化、免伺服器託管**的純前端架構，可直接於瀏覽器運行，或以 GitHub Pages 進行靜態網頁部署。

*   **核心語言**：
    *   **HTML5**：定義網頁結構、 Portal 切換及功能元件。
    *   **CSS3 (Vanilla CSS)**：控制自適應手機介面（Mobile-First）、直式 A4 報價單、以及橫式 A4 排班預覽版面。
    *   **JavaScript (ES6+)**：處理使用者互動、本機快取、排班月曆繪製、一鍵快速填入及資料運算。
*   **外部依賴套件 (透過 CDN 載入)**：
    *   `html2pdf.js (v0.10.1)`：負責將網頁區域轉換為 PDF 檔案下載。
    *   `html2canvas (v1.4.1)`：負責將 HTML 區域渲染為 Canvas 畫布，用以儲存為 PNG 圖片或進行 LINE 一鍵分享。
    *   `Google Fonts (Noto Sans TC)`：提供美觀的繁體中文字型，確保跨裝置文字排版一致。
*   **運行環境**：智慧型手機/電腦瀏覽器（如 Chrome, Safari, Edge, LINE 內建瀏覽器）。
*   **資料持久化**：使用 `localStorage`，儲存金鑰為 **`dacheng_helper_unified_state_v6`**，資料 100% 存在使用者本機設備中。

## 🔄 系統雙模組與運作機制 (System Modules & Architecture)

本系統採雙 Portal 架構，使用者可透過頂部控制列切換「報價單產生器」與「排班表產生器」。

### 系統 A：直式 A4 報價單模組 (Portrait)
*   **單向資料綁定**：編輯欄位即時寫入 `appState`，切換到預覽時重新計算並渲染。
*   **自動時數回填**：利用大直會館參數，一鍵計算當月所有週四與六日的時數及總薪資，並將完整明細回填至報價單「項目 6」。

### 系統 B：橫式 A4 雙頁排班表模組 (Landscape)
*   **動態排班表渲染**：
    *   根據所選「年份」與「月份」自動推算該月的天數（28至31天）與星期。
    *   **雙頁獨立輸出**：第一頁顯示「仁愛、安和、大直」；第二頁顯示「中山、南京」。支援「下載第一頁 PDF」、「下載第二頁 PDF」以及「下載雙頁合併完整 PDF」。
*   **新增人員與分館連動**：
    *   新增人員時，所屬館別會根據所選組別（分頁）進行下拉選單連動（組別一連動：仁愛、仁愛2、安和、大直；組別二連動：中山、南京），防止拼字錯誤。
*   **滑動日曆與手勢支援**：
    *   編輯用月曆鎖定寬度為 `480px`，並在手機端使用具有流暢物理滑動感與精美滾動條的 Scroll Wrapper 包裝，徹底解決小螢幕擠壓變形的問題。

## ⚡ 排班防呆與自動化設計 (Smart Roster Tools)

*   **🎨 與畫筆連動的快速填入工具列**：
    *   位於畫筆選擇器正下方，包含 `一` 至 `日` 七個快捷按鈕。
    *   點擊按鈕時，會**讀取當前啟用的畫筆班別**（如：上班、休假、消毒等），一鍵將當月所有對應的星期天數塗滿該班別，極大提升排班速度。
*   **💡 智慧推薦提示**：
    *   當選擇「陳淑美 (大直)」時，系統提示其固定做六日，指引點擊 `六` 和 `日` 填入。
    *   當選擇「愛玉 (大直)」時，系統提示其固定做週四，指引點擊 `四` 填入。
*   **🛡️ 同分店衝突警示 (提醒標誌顏色一致)**：
    *   當編輯某員工時，月曆日曆日期方格（高度 `72px`）下方會顯示**同分店其他同仁**當日的排班標籤。
    *   標籤的背景色、框線、文字顏色與**上方畫筆顏色完全一致**（如：藍色代表已有人上班、紅色代表已有人休假），方便排班人一眼看出當天人力分配，避免撞班。

## 🛡️ 已知技術痛點與防禦性設計 (Defensive Design Guidelines)

為了解決行動端瀏覽器與網頁渲染引擎的相容性問題，以下設計**絕對不可變動**：

### 1. 網頁載入 try...catch 安全防護
*   **痛點**：舊快取損壞或 DOM 載入時差可能導致腳本阻斷，造成頁面全白。
*   **解決方案**：在 `window.addEventListener('DOMContentLoaded')` 載入事件中，將 `loadStateFromLocalStorage()`, `initFormFields()`, `renderQuoteItemsEditor()`, `initRosterUI()` 及 `switchSystem()` 均用獨立的 `try...catch` 包裹，即使其中一個出錯，其餘功能仍可正常初始化。

### 2. 快取升級與深層驗證防覆蓋機制
*   **痛點**：使用者重新整理頁面時，若快取載入未完成就觸發自動存檔，會導致原有人員名單被覆蓋成 null。
*   **解決方案**：
    *   在記憶體宣告時預先深拷貝 `INITIAL_BRANCH_GROUPS` 以穩定 `DEFAULT_STATE`。
    *   移除了在網頁解析時即執行存檔的行為，改為在 `loadStateFromLocalStorage()` 後，才執行 `ensureRosterBranches()`。
    *   `ensureRosterBranches()` 會深層統計所有分支下的員工總數，若人數為 `0` 則自動還原預設的 22 人，並進行存檔。

### 3. PDF 下載位移與空白頁 bug
*   **痛點**：手機端滾動後下載 PDF 會導致截圖偏移出現空白。
*   **解決方案**：在 html2canvas 參數中寫死 `scrollY: 0` 和 `scrollX: 0`。
*   **痛點**：CSS 縮放（scale）導致輸出 PDF 模糊或跑版。
*   **解決方案**：建立外層縮放容器（如 `.preview-landscape-scaler`），而內層實體擷取元素（如 `#roster-page-1`）**絕不套用 transform**。
*   **痛點**：A4 Landscape 列印時因四捨五入溢出產生第二頁空白頁。
*   **解決方案**：設定 `.a4-landscape-page` 的物理寬高為高精度數值 `297mm * 209mm`（相較於 A4 標準 210mm 高度，故意縮小 1mm 安全容差），並使用 `overflow: hidden`，徹底防範分頁溢出。
